# Chris Process Rules

## Milestone Renders (MANDATORY)
After EVERY significant change, render and save a numbered screenshot:
```bash
bash render.sh fatiha-door.html fatiha-door-m1.png   # milestone 1
bash render.sh fatiha-door.html fatiha-door-m2.png   # milestone 2
```

## Progress Log
Write progress to `.chris-progress.log` after each milestone:
```
M1: [timestamp] Pencil construction lines for arch — LOOKS GOOD/BAD because...
M2: [timestamp] Added rosette pattern — LOOKS GOOD/BAD because...
```

## Self-QC Checklist (check after EVERY render)
- [ ] Is this on cream paper? (not dark, not muddy)
- [ ] Are ALL marks p5.brush strokes? (no canvas fills)
- [ ] Does it look hand-drawn? (visible grain, organic edges)
- [ ] Would I be proud to show this to Tawfeeq?
- [ ] If NO to any above: FIX before moving on

## Render Script
`render.sh` is in fatiha-proto/. Just run: `bash render.sh <file.html> <output.png>`
Server already on port 9981.

## DO NOT
- Work for more than 3 minutes without rendering
- Hand over notes about problems without fixing them
- Use ctx.fillRect(), ctx.fill(), or any canvas 2D drawing
- Exit without at least 3 milestone renders

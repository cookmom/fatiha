# Eid Mubarak Painting Brief

## Task
Paint "عيد مبارك" as a beautiful calligraphic art piece using p5.brush on cream paper.
Portrait 9:16 (1080x1920). This is for posting on X tomorrow for Eid.

## Files
- `eid-calligraphy.json` — HarfBuzz-shaped contour paths for عيد (line1) and مبارك (line2)
- `eid-ref.jpg` — Reference calligraphy style (stacked vertical, decorative, green+gold)
- `paper-texture.jpg` — Cream paper background (1080x1920)
- `mixbox.js` — Pigment-accurate color mixing (local)

## Calligraphy Data
- line1 (عيد): 4 contours, width 305 units
- line2 (مبارك): 6 contours, width 385 units
- Each contour is an array of {x,y} points (sampled bezier curves)
- Y is font-space (positive up), flip for canvas

## What Works (from v11)
- p5.brush WEBGL mode with `brush.load()` ✅
- CDNs: p5@2.0.3 + p5.brush@2.0.2-beta ✅
- Color must use `color(r,g,b)` not raw r,g,b args ✅
- `brush.set('marker', color(22,82,38), alpha)` works ✅
- Contour outline + scanline fill approach works but is UGLY — too blobby

## What's Wrong (v11 was ugly)
- Scanline fill creates horizontal banding, not organic
- Marker brush too blobby at these scales — loses letter definition
- Needs MUCH tighter, more precise brush work
- Should feel like a master calligrapher painted it, not a child with a fat marker

## Reference Style
- Look at eid-ref.jpg — the calligraphy has:
  - Deep green ink with subtle shadow/depth
  - Gold diacritical marks and decorative dots
  - Ottoman garden flowers and leaves framing it
  - Stacked vertical: عيد large on top, مبارك below
  - Elegant, precise, not blobby

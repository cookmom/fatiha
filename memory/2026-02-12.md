## WebSocket Chat — WORKING ✅
- Kitchen chat fully bidirectional — messages flow both ways with real-time streaming
- "hello from the kitchen" received and responded to successfully
- Token hardcoded in HTML (no prompt needed) — safe because gateway is loopback-only
- `deliver:true` set so responses also go to Telegram (Tawfeeq wants to see kitchen chat on phone when away)

### Key fixes that made it work:
1. Connect params: `minProtocol:3, maxProtocol:3, client:{id:'webchat-ui', version, platform:'web', mode:'webchat'}`
2. Gateway response data is in `msg.payload` NOT `msg.result` — HTML was checking wrong field
3. chat.send needs: `message` (not `text`), `sessionKey:'agent:main:main'`, `idempotencyKey`, `deliver:true`
4. Chat events: `msg.type==='event' && msg.event==='chat'`, payload has `state:'delta'|'final'|'error'`
5. Delta streaming works — text streams in real-time
6. Fallback: `fetchLastResponse()` fetches chat.history after final event if no deltas received

### Protocol reference:
- Connect: `{type:'req', method:'connect', params:{auth:{token}, minProtocol:3, maxProtocol:3, client:{id:'webchat-ui', platform:'web', mode:'webchat'}}}`
- Send: `{type:'req', method:'chat.send', params:{message:text, sessionKey:'agent:main:main', deliver:true, idempotencyKey:unique}}`
- History: `{type:'req', method:'chat.history', params:{sessionKey:'agent:main:main', limit:50}}`
- Events: payload in `msg.payload`, states: delta/final/error/aborted

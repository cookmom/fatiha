- `body{opacity:0}` in `<head>` (line 7) was being overridden by `body{opacity:1}` in main `<style>` block (line 164)
- Result: body visible from first paint, CSS animation `both` fill mode USUALLY prevents flash but can miss 1 frame on some devices
- Fix: removed `opacity:1` from main stylesheet, added `document.body.style.opacity='1'` after `buildSVG()` completes
- Body now stays invisible until splash SVG is fully constructed

### Race Condition Audit
- Confirmed no race condition between text and cube draw ‚Äî both use CSS animations from same t=0 (page parse)
- `animation-fill-mode: both` ensures text starts at opacity:0, SVG paths start with full dashoffset

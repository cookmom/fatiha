### Compass Artifact Fix (LONG battle)
- **Bug**: Thin diagonal line artifacts to RIGHT of cube in compass mode on iPhone
- **Root cause**: `_qiblaEntryBeam` (PlaneGeometry) and `_qiblaExitCaustic` (PlaneGeometry) are vertical planes inside `prismGroup` (rotation.y = PI/4). From steep camera angle (y=9.7), vertical planes show edge-on artifacts.
- **Failed attempts**:
  1. Wider plane (1.2 vs 0.06) — still edge-on ❌
  2. Shader discard for low alpha — didn't help ❌
  3. Faster fade threshold (0.7× / 0.05) — symptom fix only ❌
  4. Move to scene root — broke hide/show logic, beams showed in normal mode ❌
  5. Counter-rotate -PI/4 (billboard) — artifact moved to LEFT side ❌
- **FINAL FIX (Chris, commit f15bda1)**: Make both planes HORIZONTAL (rotation.x = -PI/2, y=0.01). Camera looks DOWN, so floor planes are always face-on. Zero edge-on risk regardless of prismGroup rotation. Entry beam = floor light strip, exit caustic = floor hotspot.
- GPU render verified: no artifacts in compass mode, normal mode clean

### HRTF Adhan — Deep Debug Session
- **Root cause of "both" at start**: mobile tap fires BOTH touchend AND mouseup → playAdhan() called twice → two HRTF buffers. Fixed with `lastTapWasTouch` flag.
- **iOS Safari ignores `volume=0`** on Audio elements — poll proved it (set 0, reads 1). Fixed with `.muted=true`.
- **iOS suspends entire page on background** — JS stops, visibility handler never fires on return sometimes, can't start new audio from background.
- **CORS blocks Internet Archive** — self-hosted adhan files in `audio/` folder (same-origin, ~1.5MB total)
- **Adhan sources now local**: `audio/adhan-mishari.mp3`, `audio/adhan-makkah.mp3`, `audio/adhan-madinah.mp3`
- **Current architecture (v21)**: HRTF only, save position on hide, restart HRTF on return. No plain Audio dual-stream.
- **Dedicated `adhanAudioCtx`** separate from `qiblaAudioCtx` (tick/alarm sounds)
- **Generation guard** (`hrtfGeneration` counter) prevents stale fetch callbacks from creating rogue buffers
- **Debug overlay**: green-on-black log panel, version tag `[vNN]`, 3s polling of all audio state
- Still testing v21 — awaiting Tawfeeq's results

### Key iOS Safari Lessons
- `Audio.volume = 0` does NOT silence on iOS — use `.muted = true`
- Can't call `new Audio().play()` from visibilitychange hidden state — iOS rejects it
- iOS suspends AudioContext AND JS execution when page goes background
- `createMediaElementSource` permanently captures Audio element — can't "unplug"
- Mobile tap fires both touchend + mouseup — always guard against double-trigger

## HRTF Spatial Adhan — SHIPPED (v22→v45, 24 iterations)
- Final architecture: pre-unlock Audio on first tap (preload only, no play) → StereoPanner with JS lerp → MediaSession guard (play only when document.hidden)
- Key bugs found & fixed:
  - Silent WAV data URI → AbortError on iOS (use real file instead)
  - Async pause events kill intervals (race condition on _swapping flag — use setTimeout)
  - HRTF PannerNode ignores live position updates on iOS (use StereoPanner instead)
  - linearRampToValueAtTime stacks and freezes after reconnect (use .value directly)
  - MediaSession play handler auto-triggers when AudioContext outputs (guard with document.hidden)
  - Pre-unlock playing real file = audible blip (preload only, unlock on actual trigger)
  - playAdhan double-call when already playing (guard with adhanPlaying flag)
- Post-SHOW panning fixed: restart interval, re-assert adhanPlaying, setTimeout on _swapping
- Gain boost 1.5x to match plain Audio loudness on iOS
- Smooth panning via JS lerp: `_currentPan += (target - current) * 0.15`

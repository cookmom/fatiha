### Ottoman Proportional System (FINAL — all from base unit U)
```
U = W/(2φ+2) ≈ 206px (base unit)
archSpan = 2U (niche width)
colWidth = U/φ² ≈ 79px
capHeight = colWidth × φ ≈ 127px
bandHeight = U/φ ≈ 127px
crownSpan = bandW × √φ (crown width)
crownR = crownSpan/√2 (sub-equilateral = wide dome-like profile)
starR = min(crownH/(2φ), crownHalfSpan/φ²)
Star center at crownPeak + crownH/φ (golden section)
Springing at golden section: compTop + compH/φ (61.8% down)
Star nested at φ ratios: r, r/φ, r/φ², r/φ³
Column entasis rings at 1/φ and 1/φ² of column height
```

### Vertical Hierarchy (bottom to top)
Ground line → Column bases → Column shafts → Capitals → Main pointed arch → Inscription band ABOVE arch peak → Crown arch (wide pointed, dome-like) → 12-fold star

### Key Design Decisions
- **Tawfeeq preferred pointed crown over dome** — but wanted wider base for dome-like feel
- Crown arch: `crownR = span/√2` gives wide base with pointed tip
- Inscription band sits ABOVE arch peak (not at springing) — this was a key fix
- Vertical jambs connect band down to capitals — nothing floats
- Blue watercolor fill opacity ≤35 so pencil strokes show through

## Animated Mihrab Drawing with HUD

### Concept
Real-time p5.brush animation drawing the Ottoman mihrab sequentially, element by element. Each element gets a HUD tracking box showing mathematical formula and live x,y coordinates.

### HUD Design (Tawfeeq-directed)
- Rectangular tracking box with **corner brackets** (L-shaped, not full border)
- Small filled squares at each corner (7×7px)
- **Dark frosted info panel INSIDE the box** (not floating outside)
- Cyan accent color (#00d4ff)
- Shows: element name (uppercase), formula derivation, live coordinates
- Pulse glow animation on element completion
- Monospace font (SF Mono / Fira Code)

### Technical Approach
- p5.brush in `draw()` loop — redraws ALL completed elements + current each frame
- HTML overlay div for HUD positioned absolutely over canvas
- Timeline-based: each step has a duration in seconds, converted to frame ranges
- Total animation ~15s at 60fps

## p5.brush v1 vs v2 Watercolor Fill Comparison (IMPORTANT)
- **v1.0.4c** (Alejandro's demo version): `brush.bleed(0.2)` produces rich, saturated watercolor bleed. Beautiful organic edges. THE GOLD STANDARD.
- **v2.0.2-beta**: `brush.fillBleed(0.2)` produces much lighter/transparent fills. Even at opacity 200, shows ugly circle/bubble artifacts. Opacity 130 is closest to v1's opacity 70 but still has artifacts.
- **v2.1.3-beta** (latest): fills barely render at all — almost completely transparent. Worse than v2.0.2.
- **Decision**: Keep v2 for real-time pencil drawing animation (draw() loop). Don't use v2 watercolor fills — they're ugly. When rich watercolor fills are needed, use v1 in a separate canvas layer.
- **API mapping**: v1 `brush.bleed(x)` → v2 `brush.fillBleed(x)`. Everything else (beginShape/vertex/endShape, fill, randomSeed) is identical.
- **Alejandro's pattern**: `noLoop()` default, `loop()` on change, draws ONE frame, `noLoop()` again. NOT continuous draw loop.
- **Fill budget**: cumulative across ALL frames in the entire animation lifetime (~43 total). NOT per-frame. This means watercolor fills in draw() loops crash after ~43 frames.

## Lessons
- p5.brush `draw()` loop clears and re-renders each frame in WEBGL — must redraw ALL previous elements each frame (no additive accumulation)
- Crown arch math: `crownR = span` (equilateral) with wide span goes way too tall, gets clamped. Use `crownR = span/√2` for dome-like profile
- Opacity stacking in p5.brush: even 80 opacity compounds to near-opaque with 3 layers. Keep ≤35 for single wash if pencil must show through
- SVG sketches (mihrab-options.html) are much faster for showing multiple design options than p5.brush renders

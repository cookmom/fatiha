## p5.brush Animation — Root Cause of Flicker (CONFIRMED)
- p5.brush's FILL system (watercolor fills) uses random polygon deformations
- Each fill() call generates DIFFERENT random shapes
- Redrawing fills every frame = different shapes every frame = flicker
- STROKES (brush.line, brush.spline, brush.flowLine) are MORE consistent but still have some randomness
- The correct architecture:
  1. Draw fills ONCE in setup() (static watercolor washes)
  2. In draw(): only animate strokes + evolving flow fields
  3. If fills need to breathe: capture as image, use Canvas2D composite with subtle transforms
- brush.refreshField(frameCount/10) is the proper way to evolve flow fields over time
- The instance mode example (background(240) + brush.line() in draw) IS the official pattern
- postdraw callback auto-composites brush strokes after each draw()

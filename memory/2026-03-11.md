### Key bugs found
1. **Live timer overwriting swipe header** — `_displayPrayerTimes` runs every second and resets `hijriTop`. Fixed with `_swipeHijriLock` boolean.
2. **SW cache 30 versions stale** — `rc-v498` was serving old JS. Always bump SW cache with significant changes.
3. **Night vs day format logic** — simplified to check prayer name (`Maghrib/Isha/Qiyam/Fajr` = night) rather than complex step counting.

### Lesson
- **Lesson 20**: When updating UI during a preview/swipe, always check if a live timer/interval will overwrite it. Add a lock flag and clear on revert.
- **Lesson 21**: SW cache (`sw.js CACHE_NAME`) must be bumped alongside major feature changes — cache-bust query params on JS files don't help if the SW serves stale HTML.

### Architecture notes
- `window._swipeHijriLock` — set in `_swipeShowPreview`, cleared in `_swipeRevert`
- `window._tomorrowPrayerTimings` — computed in `_setPrayerLocation` via `_computePrayerTimes(lat, lon, tomorrow)`
- `window._swipeShowPreview` now exposed on `window` for debugging
- Prayer classification: night=Maghrib/Isha/Qiyam/Fajr, day=Sunrise/Dhuha/Dhuhr/Asr
- Tomorrow's times used when `_stepsForward > _stepsToIsha` (past Isha in swipe sequence)
- `def.startKey` maps sector to timing key, `def.startOffset` for Dhuha (+20 from Sunrise)

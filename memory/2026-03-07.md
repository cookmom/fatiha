## Durable learnings - AGOT splash/clock/city-time debugging (added 2026-03-07 16:42 PT)

- Root bug class surfaced repeatedly: visual overlay drift and splash artifacts came from duplicated coordinate logic + timing races. Robust fix pattern is to use **single-source renderer telemetry**, then gate by stability windows.
- Countdown toggle issues were primarily event-model bugs on mobile (touch/click synthesis + rerender binding drift). Simpler first-principles interaction rule is required: one physical action => one state transition; avoid stacked handlers.
- Critical production bug discovered and fixed: prayer times were being formatted using local Date getters instead of selected-city timezone formatting in `_computePrayerTimes`; this caused wrong active prayer windows after city changes (example: Cape Town showing Dhuha at local night). Fix: format using `toLocaleTimeString(... timeZone: _cityTz)`.
- On city change, preview/simulated state must be reset to live mode (`_swipeTimeOverride`, `_swipeTimeTarget`, `_forceTimeMin`, `_swipePreviewPrayer`) or prayer windows can remain stale.

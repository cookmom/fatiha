## HRTF Spatial Adhan — Deep Dive & Decision to Strip
- Goal: play adhan spatialized toward Qibla using Web Audio HRTF PannerNode
- Multiple approaches tried over many hours:
  1. `createMediaElementSource` — permanently captures Audio element, can't switch back to plain
  2. Dual-stream (plain Audio + HRTF AudioContext) — double-play bug, both audible
  3. Buffer overlay with gain toggle — iOS AudioContext issues across visibility changes
  4. Visibility swap (kill HRTF→plain on background, restart on foreground) — timing issues
- Root problems: Internet Archive lacks CORS headers (blocking fetch+decode for buffers); iOS AudioContext suspend/resume unreliable; MediaElementSource is one-way capture
- Self-hosted adhan files fix CORS but iOS AudioContext sync issues persist
- Added extensive debug logging overlay (`adhanDbg()`, green-on-black `<pre>` panel)
- **Decision**: Strip HRTF code, ship plain Audio. Revisit when custom adhan recordings arrive (can design HRTF from scratch with known audio format)
- Plain Audio is reliable: survives backgrounding, MediaSession for lock screen, works on rerun

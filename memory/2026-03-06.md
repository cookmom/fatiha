## Late Night Session — Splash Screen Dynamic Projection (v301→v307)

### Problem
Splash wireframe cube (SVG) didn't align with 3D cube at different viewport sizes. Original v300 used hardcoded coordinates from 430×932 viewport with `preserveAspectRatio="none"` which distorted on other sizes.

### Solution (v302→v307)
- v302: Replaced hardcoded SVG with JS-computed projection using same camera params as Three.js (camPos, fov=35, lookAt)
- v303: Uniform stroke-width 1, draw order flipped (white structure first, colored violet/blue last)
- v304: Used `100lvh` for height measurement (matches Three.js canvas)
- v305: Splash div + SVG use `width:100vw; height:100lvh` (exact match with canvas CSS)
- v306: Added `_cubeScreenVerts` snap-before-fade (Tawfeeq rejected as "fudging")
- v307: Clean rewrite — Three.js exposes `window._canvasW/_canvasH`, splash auto-corrects if its initial `100lvh` measurement differs (iOS Safari URL bar). No snap hack.

### Key Insight
On iOS Safari, `100lvh` ≠ `window.innerHeight` when URL bar is visible vs hidden. PWA fullscreen has no URL bar, so viewport is taller. The splash and canvas must use the **exact same** height measurement. Three.js exposes its values, splash rebuilds if they differ.

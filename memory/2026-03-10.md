### SVG Edge Resize Fix (v463)
- **Bug**: Minimizing and returning caused SVG edges to be smaller than 3D cube
- **Root cause**: After splash, `_splashActive=false` so `tryCorrect()` stops. SVG paths frozen at splash-time viewport. 3D canvas resizes but SVG doesn't.
- **Fix**: `window._resizeEdgeOverlay()` rebuilds SVG paths from `getVerts(w,h)` at current canvas size. Called on `visibilitychange` (return from background).
- Only rebuilds cube edges (first 5 paths), not hands
